二、核心数据链路

1 管理员配置

data/admin-config.json 定义数据源、adapter、优先级、TTL、API Key 环境变量名；真实密钥由 .env 提供。

2 页面请求 API

赛程、球队、赔率、新闻、早报、雷达、球队毒舌和球员毒舌分别请求对应的 /api/data/* 聚合接口。

3 先读规范化快照

页面 API 默认 cache-only，PostgreSQL 中的页面级 payload 命中则直接返回，减少远端波动。

4 后台请求远端源

cron、初始化脚本或 worker 按 type 和 priority 过滤启用源；请求前检查原始响应数据库缓存和配额策略。

5 规范化转换

数据源转 Match[]、NewsArticle[]、RadarMatch[]、Team[] 等展示 DTO；写入快照。

6 兜底返回

远端失败时使用过期快照或官方 FIFA 赛程；没有可靠来源的数据保持为空。

四、页面到数据的加工泳道

今日赛程

远端赛程或官方兜底经过统一 Match 模型后进入页面。

远端赛程源 / FIFA JSON 缓存与快照 日期过滤、北京时间格式化、队名映射 MatchCard / HeroSpotlight 失败则本地兜底
天眼雷达

预测市场数据被筛选成世界杯相关事件，再与赔率赛事匹配并比较隐含概率。

Polymarket events 原始响应缓存 筛选 World Cup/FIFA，解析 outcomePrices，与赔率匹配 差值、趋势、标签展示 缺少真实赔率则为空
早报 / 详情 / 球队

早报由多路真实新闻源并行抓取，经过规则去重后交给管理员选择的主 AI Provider 整理；球队和球员毒舌也由后台快照生成，失败时使用规则兜底。

ESPN / ChinaNews / BBC / Currents / GDELT / NewsAPI URL 与标题相似度去重 管理员选择的主模型 其他启用模型备用 无数据或全失败则为空
工具箱

完全浏览器本地计算，没有网络请求。

用户输入 1 / 欧赔 × 100% 期望收益公式 打平命中率 风险解释文案

五、AI 生成的实际位置

当前代码里：已接入可选后台 AI 整理链路

新闻聚合完成规则去重后，会优先调用管理员选择且已启用的主 AI Provider，输出结构化事件摘要、要点、早报标题和总摘要；失败时按配置顺序自动尝试其他已启用 Provider。当前默认主模型配置为 NVIDIA NIM deepseek-ai/deepseek-v4-pro，读取 NVIDIA_API_KEY，DeepSeek 和 Xiaomi MiMo 可作为备用。

安全边界：事实先行，生成层可降级

AI 输入只包含规范化后的标题、摘要、来源、发布时间、球队资料和赛程上下文，并被要求不能补充未提供事实。模型失败不会阻断接口，也不会用本地 mock 替代。

结论：当前内容体系以真实数据源和 FIFA 官方赛程快照为基础。没有可靠来源的数据保持为空；AI 只适合作为事实数据之后的表达层。

六、当前架构风险点

WorldCupAPI 权限未开通

密钥已保存在本地配置，但供应商返回账号无数据访问权限，因此保持禁用，避免进入失败重试链路。

雷达事件匹配待完善

赔率数据已接入；雷达仍需把 Polymarket 事件与赔率事件可靠对齐，匹配前不生成对照记录。

球队深层内容仍为空

48 队基础资料、阵容和毒舌快照已接入；仍需继续补强可审计来源、伤病和最新名单字段。
